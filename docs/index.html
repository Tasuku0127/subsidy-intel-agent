Problem

なぜこのツールが必要か？

😰 現状の人力運用

  • ✗
    更新漏れ
    補助金の期限・金額・対象条件が古いまま広告・LPで訴求し続ける
  • ✗
    検知の遅延
    「気づいたら締切が変わっていた」→ 広告費のムダ、機会損失
  • ✗
    修正作業の属人化
    誰が・どのページの・どこを直すかが曖昧、対応スピードが遅い

📉 ビジネスへの影響

CVR 低下
古い制度情報でユーザーの信頼を失う
CPA 悪化
終了した補助金で広告を配信し続ける
制作工数の浪費
情報収集〜修正案作成に数時間かかる

再エネ・電力・脱炭素領域の特殊事情： 補助金・制度・申請期間・要件変更の頻度が高く、LP/記事/広告訴求の「鮮度」がCVと信頼性に直結する。 国の政策目標（2030年 ZEH標準化等）により毎年制度が大きく変わる。

Solution

このツールが実現すること

検知 → 分析 → 提案 の 3ステップを自動化

🔍

① 自動検知

YAML で指定した補助金ページを定期クロール。 前回スナップショットとの差分を段落単位で検出。 金額・日付・重要語に重み付けしてfalse positiveを抑制。

数時間→数分 で変更を把握
🏷️

② 変更分類

変更タイプを自動判定。 締切変更 金額改定 対象変更 手続き変更 など。 重要度（Severity）を3段階で自動スコアリング。

🔴 HIGH は即時アラート
✍️

③ 修正案生成

Claude API が差分を読み取り、 自社LP修正文案と 広告訴求案（RSA形式）を自動生成。 確認ポイントも列挙。根拠差分を常に引用。

制作工数を大幅削減
